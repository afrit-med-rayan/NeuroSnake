## рҹ§  Overview

![Gameplay Screenshot](docs/assets/gameplay.png)

**NeuroSnake** teaches a neural network to play Snake from absolute zero вҖ” no hardcoded rules, no lookahead algorithms. The agent observes an 11-feature state vector, picks actions using a Deep Q-Network, and learns purely through trial and error guided by a sparse reward signal.

Training leverages two core DRL stabilisation techniques:
- **Experience Replay** вҖ” random mini-batches break temporal correlations in training data
- **Target Network** вҖ” a periodically-frozen copy of the Q-network prevents oscillating Q-value updates

After ~300вҖ“600 episodes the agent reliably collects food and avoids walls. After 1 000 episodes it develops efficient routing strategies entirely on its own.

The agent picks the action with the highest Q-value during exploitation, or a random action during exploration (Оө-greedy).

### Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| State representation | 11 binary features | Fast, efficient вҖ” no CNN overhead |
| Action space | 3 (straight / right / left) | Relative directions eliminate illegal 180В° reversals |
| NN architecture | 11 вҶ’ 128 вҶ’ 256 вҶ’ 3 | Sufficient capacity; no overfitting on compact state |
| Exploration | Оө-greedy (exponential decay) | Standard, proven for tabular-to-DQN transfer |
| Stability | Target network (frozen copy) | Breaks the moving-target problem in Q-learning |
| Optimizer | Adam (lr = 0.001) | Adaptive learning rate вҖ” robust across hyperparameter ranges |
| Discount factor | Оі = 0.99 | High future-reward weight encourages survival |

## рҹ“Ў State Representation (11 Features)

The agent never sees raw pixels. Instead, each game frame is encoded as **11 binary floats** вҖ” a compact but complete description of the snake's immediate situation.

```
State vector  =  [ dangerГ—3 | directionГ—4 | foodГ—4 ]
```

| Index | Category | Feature | Value |
|-------|----------|---------|-------|
| `0` | вҡ пёҸ Danger | **Straight** вҖ” collision one step ahead in current direction | 0 / 1 |
| `1` | вҡ пёҸ Danger | **Right** вҖ” collision one step to the right (clock-wise) | 0 / 1 |
| `2` | вҡ пёҸ Danger | **Left** вҖ” collision one step to the left (counter-CW) | 0 / 1 |
| `3` | рҹ§ӯ Direction | Moving **RIGHT** | 0 / 1 |
| `4` | рҹ§ӯ Direction | Moving **DOWN** | 0 / 1 |
| `5` | рҹ§ӯ Direction | Moving **LEFT** | 0 / 1 |
| `6` | рҹ§ӯ Direction | Moving **UP** | 0 / 1 |
| `7` | рҹҚҺ Food | Food is to the **LEFT** of head | 0 / 1 |
| `8` | рҹҚҺ Food | Food is to the **RIGHT** of head | 0 / 1 |
| `9` | рҹҚҺ Food | Food is **ABOVE** head | 0 / 1 |
| `10` | рҹҚҺ Food | Food is **BELOW** head | 0 / 1 |

> All features are `float32`. The direction block is one-hot (exactly one of indices 3вҖ“6 is `1`). The danger and food blocks can be multi-hot.

**Why only 11 features?**  
A CNN reading raw pixels needs millions of parameters and hours of GPU training. An 11-feature hand-crafted state lets a tiny 3-layer MLP learn effective play in under 20 minutes on CPU вҖ” no GPU required.

## рҹҸҶ Reward System

The reward signal is deliberately sparse and simple вҖ” the agent must figure out everything else through Q-learning.

| Event | Reward | Rationale |
|-------|--------|-----------|
| рҹҚҺ Food collected | **+10.0** | Strong positive signal вҖ” the primary objective |
| рҹ’Җ Wall or self collision | **вҲ’10.0** | Strong negative signal вҖ” death is catastrophic |
| вҸұпёҸ Each step taken | **вҲ’0.1** | Small penalty discourages aimless wandering and infinite loops |
| вҸ° Timeout (> max steps) | **вҲ’10.0** | Treated as death вҖ” prevents the snake from spinning forever |

> **Max steps per episode** = `GRID_SIZE Г— GRID_SIZE Г— 2` = 800 steps on the default 20Г—20 grid.

## рҹҡҖ Usage

The project is driven by a single unified CLI entry point: `main.py`.

### 1. Train the Agent

```bash
python main.py train
```
Watch the AI learn from scratch. A Pygame window displays the agent's actions, while a live Matplotlib chart plots the score and the rolling 100-episode mean.

![Training Curve](docs/assets/training_curve.png)

- The model automatically saves the best run to `models/dqn_snake.pth`.
- Training will run for `NUM_EPISODES` (configurable in `config.py`).

### 2. Play Manually

```bash
python main.py human
```
Play the game yourself using **Arrow Keys** or **W A S D**. Press **R** to restart after game over, or **ESC** to quit.

### 3. Evaluate the Agent

```bash
python main.py eval
```
Loads the saved `models/dqn_snake.pth` weights and watches the agent play. 
- Exploration is completely disabled (`Оө = 0`).
- The agent plays greedily using the learned Q-values.

## рҹ“Ҡ Expected Results

1. **0вҖ“100 episodes**: The agent moves semi-randomly, frequently crashing into walls and exploring the space.
2. **100вҖ“300 episodes**: The agent learns to avoid walls. The average score starts to climb.
3. **300вҖ“600 episodes**: The agent reliably tracks the food. High scores will exceed 30вҖ“40.
4. **600+ episodes**: The agent develops routing strategies to avoid getting trapped by its own tail.

## рҹ”® Future Improvements

While this DQN agent plays successfully, it does not achieve "perfect play" (filling the entire board). Potential avenues for future iteration:

- **CNN + Frame Stacking:** Replace the 11-feature state with 4 stacked pixel frames and use Convolutional layers.
- **Double DQN (DDQN):** Decouple action selection from action evaluation to reduce overestimation bias in Q-values.
- **Prioritized Experience Replay (PER):** Sample experiences with high TD-error more frequently to speed up learning.
- **Dueling DQN:** Separate the estimation of state value `V(s)` and action advantage `A(s, a)` to stabilize learning when actions don't highly influence the state value.
